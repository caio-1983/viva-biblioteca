# Regras para Claude

Antes de implementar qualquer funcionalidade:

1. Ler toda a pasta docs.
2. Não alterar regras de negócio sem autorização.
3. Não criar autenticação.
4. Não utilizar Supabase.
5. Não utilizar PostgreSQL.
6. Utilizar SQLite.
7. Utilizar Prisma.
8. Seguir a arquitetura Service → Repository → Prisma.
9. Seguir padrões descritos em docs/.
10. Sempre atualizar documentação quando criar novas funcionalidades.
